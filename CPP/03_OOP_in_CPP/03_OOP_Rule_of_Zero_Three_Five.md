# OOP in C++ -- Rule of Zero, Three, & Five

## 1. The Five Special Member Functions

C++ classes can automatically generate or user-define five special member functions:

1. **Destructor**: `~Widget()`
2. **Copy Constructor**: `Widget(const Widget&)`
3. **Copy Assignment Operator**: `Widget& operator=(const Widget&)`
4. **Move Constructor** *(C++11)*: `Widget(Widget&&)`
5. **Move Assignment Operator** *(C++11)*: `Widget& operator=(Widget&&)`

---

## 2. Rule of Three (Pre-C++11)

If a class manages raw resource allocations (like a raw pointer on the heap), you must write a custom **Destructor** to free it. 
If you do, the default copy semantics generated by the compiler will perform a shallow copy, leading to double-free crashes. Thus, you must define:
1. **Destructor** (to free memory)
2. **Copy Constructor** (to perform deep copy)
3. **Copy Assignment Operator** (to perform deep copy safely)

---

## 3. Rule of Five (Modern C++)

With C++11 move semantics, if you write a custom destructor or copy operations, you should also write or declare the move operations to enable optimization and prevent the compiler from generating copy operations in places where moves are possible.

```cpp
#include <iostream>
#include <cstring>
#include <utility>

class Buffer {
    char* data;
    size_t size;
public:
    Buffer(size_t sz) : size(sz) {
        data = new char[size];
    }

    // 1. Destructor
    ~Buffer() {
        delete[] data;
    }

    // 2. Copy Constructor (Deep Copy)
    Buffer(const Buffer& other) : size(other.size) {
        data = new char[size];
        std::memcpy(data, other.data, size);
    }

    // 3. Copy Assignment Operator (Copy and Swap Idiom)
    Buffer& operator=(const Buffer& other) {
        if (this != &other) {
            Buffer temp(other);
            std::swap(data, temp.data);
            std::swap(size, temp.size);
        }
        return *this;
    }

    // 4. Move Constructor (Resource Theft)
    Buffer(Buffer&& other) noexcept : data(nullptr), size(0) {
        data = other.data;
        size = other.size;
        
        other.data = nullptr; // Null out source to prevent double-free
        other.size = 0;
    }

    // 5. Move Assignment Operator
    Buffer& operator=(Buffer&& other) noexcept {
        if (this != &other) {
            delete[] data; // Free existing resource
            
            data = other.data;
            size = other.size;
            
            other.data = nullptr;
            other.size = 0;
        }
        return *this;
    }
};
```

---

## 4. Rule of Zero

> [!TIP]
> The **Rule of Zero** states that classes should not define any of the five special member functions if they can rely on standard resource managers (like `std::string`, `std::vector`, `std::unique_ptr`).

By avoiding raw pointers and using modern smart resource classes, the compiler generates all copy/move operations correctly and safely without any custom boilerplate code.

```cpp
#include <string>
#include <vector>

// Rule of Zero compliant class: no custom destructor/copy/move needed!
class Student {
    std::string name;
    std::vector<int> grades;
};
```

## 5. Explicitly Defaulting and Deleting (`= default`, `= delete`)

- **`= default`**: Instructs the compiler to generate the default implementation (even if other constructors would have suppressed it).
- **`= delete`**: Prevents the compilation of specific functions (e.g. making a class non-copyable).

```cpp
class NonCopyable {
public:
    NonCopyable() = default; // force compiler default constructor
    
    // Disable copy constructors
    NonCopyable(const NonCopyable&) = delete;
    NonCopyable& operator=(const NonCopyable&) = delete;
};
```
